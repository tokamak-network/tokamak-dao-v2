# vTON 발행량 통제 방안 비교 분석

> 작성일: 2026-02-05

## 배경

vTON은 현재 **무제한 발행 가능**한 구조이며, `emissionRatio`(0~100%)를 owner(향후 DAO)가 수동으로 조정한다. 발행량을 통제하는 "축"을 무엇으로 두느냐에 따라 두 가지 후보안이 제시되었다.

### 현재 구조

| 항목 | 현재 구현 |
|------|----------|
| 발행 방식 | 인가된 minter의 `mint()` 호출 (pull-based) |
| Emission Ratio | 0~100%, 초기값 100% (1e18) |
| 비율 조정 권한 | Owner-only (`setEmissionRatio`) |
| Rate Limiting | 없음 |
| Supply Cap | 없음 |
| TON 스테이킹 연동 | 컨트랙트 레벨에서 미구현 |

## 안 1: 시간 축 - 반감기 모델 (Kevin 안)

### 개요

비트코인 반감기처럼 일정 블록 수(약 6개월)마다 TON 대비 vTON 발행 비율을 자동으로 50%씩 감소시킨다.

- 1기: 100% → 2기: 50% → 3기: 25% → 4기: 12.5% → ...

### 장점

| 항목 | 설명 |
|------|------|
| 예측 가능성 | 발행 스케줄이 확정적. 투자자와 참여자 모두 미래 희석률을 사전에 계산 가능 |
| 거버넌스 오버헤드 최소화 | 비율 조정을 위한 반복적인 DAO 투표가 불필요. 코드가 곧 정책 |
| 조작 저항성 | 고래나 이해집단이 발행률을 자기에게 유리하게 변경할 수 없음 |
| 구현 단순성 | 컨트랙트에 `deployBlock` + `halvingInterval` + 현재 블록 기반 계산만 추가 |
| 신뢰성 (Credible Neutrality) | 비트코인이 증명한 모델. "규칙은 배포 시점에 확정"이라는 강력한 사회적 계약 |

### 단점

| 항목 | 설명 |
|------|------|
| 경직성 | 네트워크가 급성장하거나 위기 상황에서도 비율 조정 불가 |
| 초기 과잉발행 위험 | 초기 1:1 구간에서 대량 발행 후 이탈하는 "초기 채굴자 문제" 발생 가능 |
| 네트워크 상태 무시 | 밸리데이터가 10명이든 1,000명이든 동일한 비율 적용. 인센티브 미스매치 가능 |
| 비가역성 | 반감기가 너무 빨랐다고 판단되어도 되돌릴 수 없음 (하드포크 수준 변경 필요) |
| DAO 무력화 | 발행 정책에 대한 거버넌스 권한을 사실상 포기하는 것 |

## 안 2: 상태 축 - 네트워크 반응형 모델 (Thomas 안)

### 개요

밸리데이터 수, 스테이킹 비율 등 온체인 상태에 따라 DAO가 `emissionRatio`를 유동적으로 조정한다.

### 장점

| 항목 | 설명 |
|------|------|
| 적응성 | 네트워크 성장 단계에 맞는 인센티브 설계 가능 |
| 목적 정합성 | "밸리데이터를 더 확보해야 한다" → 발행 확대, "충분하다" → 발행 축소. 수단과 목적이 일치 |
| DAO 주권 유지 | 토큰홀더가 통화정책에 대한 결정권을 보유. 거버넌스의 존재 이유에 부합 |
| 위기 대응 가능 | 네트워크 위축 시 인센티브를 높여 밸리데이터 이탈을 방지할 수 있음 |
| 점진적 탈중앙화 | 초기에는 적극적 조정, 성숙 후에는 비율을 낮추는 등 단계적 전환 가능 |

### 단점

| 항목 | 설명 |
|------|------|
| 거버넌스 공격 벡터 | vTON 대량 보유자가 발행 비율을 높이는 제안을 통과시켜 자기 이익 극대화 가능 |
| 예측 불가능성 | 미래 발행량을 예측할 수 없어 토큰 가치 평가가 어려움 |
| 거버넌스 피로 | 비율 조정을 위한 반복적인 투표가 필요. 참여율 저하 시 소수 의결 리스크 |
| 오라클 의존성 | 밸리데이터 수 등 상태 데이터를 온체인에서 신뢰성 있게 가져와야 함 |
| 의사결정 지연 | 급변하는 상황에서 프로포절 → 투표 → 실행까지 최소 수일~수주 소요 |

## 하이브리드 안 (검토용)

두 안이 반드시 배타적일 필요는 없다. 예측 가능성(시간 축)과 적응성(상태 축)을 동시에 확보하는 하이브리드 접근을 고려할 수 있다.

1. **기본 궤도는 반감기 (Kevin):** 디폴트 감소 스케줄을 컨트랙트에 내장
2. **DAO 오버라이드 권한 (Thomas):** DAO 투표로 기본 궤도 대비 ±N% 범위 내에서 조정 가능
3. **하드 캡 설정:** 총 발행량 상한 또는 연간 발행률 상한을 두어 극단적 시나리오 방지

현재 컨트랙트의 `emissionRatio` 구조는 하이브리드 모델로의 확장이 가능하다. `setEmissionRatio`를 단순 setter에서 "기본 반감기 스케줄 ± DAO 조정폭" 로직으로 변경하면 된다.
